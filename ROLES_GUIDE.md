# Руководство по системе ролей

## Обзор

Система ролей позволяет управлять доступом пользователей к различным функциям бота и ограничивать доступ к складам по группам. Вход в систему осуществляется через логин и пароль.

## Вход в систему

При первом запуске бота `/start` пользователю предлагается ввести:
1. **Логин** - уникальное имя пользователя
2. **Пароль** - секретный пароль

После успешного входа создается сессия, которая сохраняется до выхода из бота.

### Данные по умолчанию
- **Логин**: admin
- **Пароль**: admin123

⚠️ Рекомендуется сменить пароль после первого входа!

## Роли пользователей

### 👑 Управляющий (admin)
- **Полный доступ** ко всем функциям бота
- Может управлять пользователями (добавлять, удалять, изменять роли)
- Видит все склады независимо от групп
- Может управлять всеми справочниками (товары, фирмы, клиенты, цены, коалиции)

### 📊 Менеджер (manager)
- Может управлять товарами (приход/вывод)
- Доступ к отчетам и истории
- Может просматривать остатки
- Может управлять ценами
- Доступ только к складам своей группы (если назначена)

### 🏢 Завсклад (warehouse_manager)
- Все права менеджера
- Может управлять складами своей группы
- Может управлять товарами на складах
- Доступ только к складам своей группы

### 💰 Кассир (cashier)
- Только просмотр остатков
- Просмотр истории операций
- Может выполнять вывод товара (продажи)
- Доступ только к складам своей группы

## Группы складов

Группы складов позволяют ограничить доступ пользователей к определенным складам:

- **Админ** видит все склады
- **Остальные роли** видят только склады своей группы
- Пользователь без группы не видит никаких складов (кроме админа)

## Управление пользователями

### Добавление пользователя

1. Перейдите в меню "👥 Пользователи"
2. Выберите "➕ Добавить пользователя"
3. Введите логин (минимум 3 символа)
4. Введите пароль (минимум 4 символа)
5. Введите имя пользователя (ФИО)
6. Выберите роль
7. Если роль не "Управляющий", выберите группу складов

### Изменение роли

1. Перейдите в меню "👥 Пользователи"
2. Выберите "✏️ Изменить роль"
3. Выберите пользователя из списка
4. Выберите новую роль

### Назначение группы складов

1. Перейдите в меню "👥 Пользователи"
2. Выберите "🏢 Назначить склад"
3. Выберите пользователя из списка
4. Выберите группу складов

### Деактивация/Активация

- **Деактивация**: пользователь не сможет войти в бот
- **Активация**: восстановление доступа

### Удаление пользователя

Полное удаление пользователя из системы.

## Логи доступа

Все действия пользователей записываются в таблицу `access_logs` для аудита:
- ID пользователя
- Действие
- Детали
- Время

## Миграция данных

При первом запуске системы ролей:
1. Все существующие админы автоматически переносятся в таблицу `users` с ролью "admin"
2. Для всех админов создается логин "admin" и пароль "admin123"
3. Создается таблица сессий для хранения активных входов

## Примеры использования

### Пример 1: Региональные склады
```
Группа "Север":
  - Склад А
  - Склад Б
  
Группа "Юг":
  - Склад В
  - Склад Г

Завсклад Иван (группа "Север") - видит только склады А и Б
Завсклад Петр (группа "Юг") - видит только склады В и Г
Админ - видит все склады
```

### Пример 2: Иерархия доступа
```
Управляющий → полный доступ
Завсклад → управление складом + товары
Менеджер → только товары
Кассир → только просмотр и продажи
```

## Безопасность

- Пароли хранятся в виде SHA-256 хеша
- Пользователи без логина/пароля не могут войти в бот
- Деактивированные пользователи не могут войти в бот
- Только админы могут управлять пользователями
- Все действия логируются
- Сессии автоматически обновляются при активности
