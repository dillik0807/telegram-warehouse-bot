# История изменений

## Версия 2.0 - Система ролей и аутентификация

### Добавлено
- 🔐 **Система логин/пароль** вместо Telegram ID
  - Вход через логин и пароль
  - Хеширование паролей (SHA-256)
  - Управление сессиями
  - Автоматическое обновление активности

- 👥 **Система ролей** с 4 уровнями доступа:
  - 👑 Управляющий (admin) - полный доступ
  - 📊 Менеджер (manager) - управление товарами
  - 🏢 Завсклад (warehouse_manager) - управление складом
  - 💰 Кассир (cashier) - просмотр и продажи

- 🏢 **Привязка к группам складов**:
  - Пользователи видят только склады своей группы
  - Админ видит все склады
  - Гибкое назначение групп

- 🚪 **Кнопка "Выход"** в главном меню:
  - Выход из системы
  - Удаление сессии
  - Логирование выхода

- 📊 **Управление коалициями**:
  - Добавление коалиций
  - Просмотр списка
  - Удаление коалиций

- 📝 **Логирование действий**:
  - Таблица access_logs
  - Запись всех действий пользователей
  - Аудит системы

### Изменено
- Главное меню: заменена кнопка "❌ Отмена" на "🚪 Выход"
- Добавление пользователей: теперь через логин/пароль вместо ID
- Валюта: все цены в долларах ($) вместо сумов
- Меню "Пользователи": расширенное управление с ролями

### Миграции
- `migrate_db.py` - основные таблицы
- `migrate_roles.py` - система ролей
- `migrate_login_system.py` - логин/пароль

### Документация
- `LOGIN_GUIDE.md` - инструкция по входу
- `ROLES_GUIDE.md` - описание ролей
- `CHANGELOG.md` - история изменений

- Пароль: `admin123`

## Версия 2.1 - Редактирование и многопользовательская работа

### Добавлено
- ✏️ **Полное редактирование записей** (только для админов):
  - Редактирование всех полей прихода (дата, вагон, товар, источник, количество, примечания)
  - Редактирование всех полей расхода (дата, товар, количество, цена, примечания)
  - Редактирование погашений (дата, сомони, курс, примечания)
  - Редактирование партнеров (дата, сомони, курс, примечания)
  - Автоматический пересчет сумм в $ при изменении сомони или курса
  - Показ полной информации о записи перед редактированием
  - Выбор конкретного поля для изменения

- 🔒 **Защита от конфликтов при многопользовательской работе**:
  - Блокировки на уровне базы данных (SELECT FOR UPDATE)
  - Защита операций с товарами от race conditions
  - Безопасные транзакции с автоматическим откатом
  - Изоляция данных пользователей
  - Корректная работа при одновременном доступе нескольких пользователей

### Изменено
- Функция редактирования: теперь можно изменять все поля, а не только примечания
- Функции базы данных: добавлены блокировки для предотвращения конфликтов
  - `add_product()` - с блокировкой строки
  - `remove_product()` - с блокировкой строки
  - `add_arrival()` - с блокировкой строки товара
  - `add_departure()` - с блокировкой строки товара

### Исправлено
- Race conditions при одновременном добавлении/выводе товара
- Потенциальные конфликты при работе нескольких пользователей
- Некорректные остатки при одновременных операциях

### Документация
- `CONCURRENCY_GUIDE.md` - подробное руководство по многопользовательской работе (EN)
- `МНОГОПОЛЬЗОВАТЕЛЬСКАЯ_РАБОТА.md` - краткая инструкция (RU)

### Технические детали
- Использование PostgreSQL блокировок (FOR UPDATE)
- Уровень изоляции транзакций: READ COMMITTED
- Автоматический откат при ошибках
- Логирование всех операций с указанием пользователя
