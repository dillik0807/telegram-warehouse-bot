# ✅ Добавлено управление товарами

## Проблема
В разделе "Управление товарами" не было возможности добавлять товары через веб-интерфейс.

## Решение
Добавлена форма для добавления товаров и кнопка удаления.

## 📦 Управление товарами

### Добавлено
- ➕ Кнопка "Добавить товар"
- Модальное окно с формой:
  - Название товара (обязательно)
  - Подсказка о вводе уникального названия
- 🗑 Кнопка удаления для каждого товара
- Проверка существования перед добавлением
- Проверка использования перед удалением

### Как работает добавление товара

Товар добавляется через создание системной записи в таблице `arrivals`:
```sql
INSERT INTO arrivals 
(arrival_date, wagon_number, firm_id, warehouse_id, product_name, 
 source, quantity_doc, quantity_actual, notes, created_by)
VALUES 
(CURRENT_DATE, 'SYSTEM', 1, 1, 'Название товара', 
 'Добавлено вручную', 0, 0, 'Товар добавлен через веб-интерфейс', user_id)
```

**Особенности**:
- `wagon_number = 'SYSTEM'` - маркер системной записи
- `quantity_doc = 0` и `quantity_actual = 0` - нулевое количество
- Товар сразу появляется в списках выбора при создании приходов/расходов

### Как работает удаление товара

**Проверка использования**:
```sql
SELECT COUNT(*) FROM arrivals 
WHERE product_name = 'Название' AND wagon_number != 'SYSTEM'
```

**Логика**:
1. Проверяется, используется ли товар в реальных записях (не системных)
2. Если используется - показывается ошибка с количеством записей
3. Если не используется - удаляются ВСЕ записи товара (включая системные)

### Преимущества подхода

✅ **Простота**: Не нужна отдельная таблица для товаров
✅ **Совместимость**: Работает с существующей структурой БД
✅ **Автоматизация**: Товары из приходов автоматически появляются в списке
✅ **Гибкость**: Можно добавлять товары заранее, до создания приходов

## 📊 Маршруты

### GET /management/products
Отображает список всех товаров

### POST /management/products/add
Добавляет новый товар
- **Параметры**: `product_name`
- **Проверки**: Существование товара
- **Действие**: Создание системной записи в `arrivals`

### GET /management/products/delete/<product_name>
Удаляет товар
- **Параметры**: `product_name` (в URL)
- **Проверки**: Использование в реальных записях
- **Действие**: Удаление всех записей товара

## 🎯 Использование

### Добавление товара
1. Перейдите в "⚙️ Управление" → "📦 Товары"
2. Нажмите "Добавить товар"
3. Введите название товара (например: "Навои Карбамид")
4. Нажмите "Добавить"
5. Товар появится в списке и будет доступен при создании приходов

### Удаление товара
1. Найдите товар в списке
2. Нажмите кнопку удаления (🗑)
3. Подтвердите действие
4. Если товар не используется - он будет удален
5. Если используется - появится сообщение с количеством записей

## ⚠️ Важно

1. **Уникальность**: Название товара должно быть уникальным
2. **Системные записи**: Товары, добавленные вручную, имеют маркер `wagon_number = 'SYSTEM'`
3. **Удаление**: Нельзя удалить товар, который используется в реальных приходах
4. **Автоматическое добавление**: Товары также добавляются автоматически при создании приходов

## ✅ Результат

Теперь администраторы могут:
- ✅ Добавлять товары заранее, до создания приходов
- ✅ Видеть полный список всех товаров в системе
- ✅ Удалять неиспользуемые товары
- ✅ Управлять номенклатурой товаров
- ✅ Подготавливать систему к новым поставкам

## 🔄 Два способа добавления товаров

### 1. Вручную через управление
- Перейти в "Управление" → "Товары"
- Нажать "Добавить товар"
- Ввести название
- Товар сразу доступен для использования

### 2. Автоматически через приходы
- Создать новый приход
- Ввести название товара в поле "Товар"
- Товар автоматически добавится в систему
