# Руководство по управлению пользователями

## Доступ к управлению

Управление пользователями доступно только **администраторам**.

### Как открыть:
1. **Через навигацию**: Меню слева → "👥 Пользователи"
2. **Через главную страницу**: Раздел "Быстрое управление" → "👥 Пользователи"
3. **Через управление**: Меню "⚙️ Управление" → Карточка "Пользователи"

## Функции управления

### 1. Просмотр списка пользователей

На странице отображается таблица со всеми пользователями:
- **ID** - Telegram ID пользователя
- **Логин** - Логин для входа в веб-приложение
- **Имя** - Имя пользователя
- **Роль** - Роль в системе (Администратор, Менеджер, Завсклад)
- **Группа складов** - Назначенная группа складов (если есть)
- **Статус** - Активен или Неактивен

### 2. Добавление пользователя

**Кнопка:** "➕ Добавить пользователя" (в правом верхнем углу)

**Необходимые данные:**
- **Telegram ID** - ID пользователя в Telegram (обязательно)
- **Имя пользователя** - Полное имя (обязательно)
- **Логин** - Логин для входа в веб-приложение (обязательно)
- **Пароль** - Пароль для входа (обязательно)
- **Роль** - Выбор из списка (обязательно):
  - Менеджер
  - Завсклад
  - Администратор
- **Группа складов** - ID группы складов (опционально)

**Примечания:**
- Если не указать группу складов, пользователь будет иметь доступ ко всем складам
- Пароль хранится в зашифрованном виде (SHA-256)
- Логин должен быть уникальным

### 3. Активация/Деактивация пользователя

**Деактивация:**
- Кнопка "⏸ Деактивировать" для активных пользователей
- Деактивированный пользователь не может войти в систему
- Данные пользователя сохраняются

**Активация:**
- Кнопка "▶ Активировать" для неактивных пользователей
- Восстанавливает доступ к системе

**Когда использовать:**
- Временное отключение доступа (отпуск, больничный)
- Блокировка подозрительной активности
- Перед удалением (для проверки последствий)

### 4. Удаление пользователя

**Кнопка:** "🗑 Удалить" (красная кнопка)

**Внимание:**
- ⚠️ Действие необнеобратимо!
- Удаляются все данные пользователя
- История операций, созданных пользователем, сохраняется
- Требуется подтверждение

**Рекомендация:**
Сначала деактивируйте пользователя и убедитесь, что это не повлияет на работу системы.

## Роли пользователей

### Администратор (admin)
- Полный доступ ко всем функциям
- Управление пользователями
- Управление складами и клиентами
- Просмотр всех данных
- Добавление/редактирование/удаление записей

### Менеджер (manager)
- Добавление приходов и расходов
- Добавление погашений и партнеров
- Просмотр отчетов
- Доступ к назначенным складам
- Нет доступа к управлению

### Завсклад (warehouse)
- Добавление приходов и расходов
- Просмотр отчетов
- Доступ только к своим складам
- Ограниченный функционал

## Группы складов

Группы складов позволяют ограничить доступ пользователя к определенным складам.

**Как работает:**
- Если группа не указана → доступ ко всем складам
- Если группа указана → доступ только к складам этой группы

**Примеры использования:**
- Завсклад видит только свой склад
- Менеджер региона видит склады своего региона
- Администратор видит все склады

## Безопасность

### Пароли
- Хранятся в зашифрованном виде (SHA-256)
- Нельзя восстановить забытый пароль (только создать новый)
- Рекомендуется использовать сложные пароли

### Логины
- Должны быть уникальными
- Используются для входа в веб-приложение
- Не совпадают с Telegram ID

### Telegram ID
- Используется для связи с Telegram ботом
- Должен быть уникальным
- Можно узнать через бота (@userinfobot)

## Частые вопросы

**Q: Как узнать Telegram ID пользователя?**
A: Пользователь может узнать свой ID через бота @userinfobot в Telegram.

**Q: Можно ли изменить роль пользователя?**
A: В текущей версии нужно удалить и создать заново. В будущем будет функция редактирования.

**Q: Что делать, если пользователь забыл пароль?**
A: Администратор должен удалить пользователя и создать нового с новым паролем.

**Q: Можно ли восстановить удаленного пользователя?**
A: Нет, удаление необратимо. Нужно создать нового пользователя.

**Q: Сколько администраторов может быть?**
A: Неограниченно, но рекомендуется минимум 2 для безопасности.

## Рекомендации

1. **Регулярный аудит**: Проверяйте список пользователей раз в месяц
2. **Минимальные права**: Давайте только необходимые права
3. **Деактивация**: Деактивируйте уволенных сотрудников сразу
4. **Резервный админ**: Всегда должно быть минимум 2 администратора
5. **Документирование**: Ведите список пользователей отдельно
6. **Сложные пароли**: Требуйте использования сложных паролей
7. **Обучение**: Обучайте новых пользователей перед выдачей доступа

## Устранение проблем

**Пользователь не может войти:**
- Проверьте, активен ли пользователь
- Проверьте правильность логина и пароля
- Проверьте, не истекла ли сессия

**Пользователь не видит свои склады:**
- Проверьте назначенную группу складов
- Убедитесь, что склады привязаны к правильной группе

**Не могу добавить пользователя:**
- Проверьте уникальность логина
- Проверьте уникальность Telegram ID
- Убедитесь, что все обязательные поля заполнены
