# Руководство по удалению записей (для администраторов)

## Доступ к функциям удаления

Функции удаления доступны **только администраторам**.

Кнопки удаления отображаются в колонке "Действия" в следующих разделах:
- 📥 Приходы
- 📤 Расходы
- 💰 Погашения
- 🤝 Партнеры
- 👥 Пользователи

## Как удалить запись

### Шаг 1: Открыть нужный раздел
Перейдите в раздел с записями, которые нужно удалить:
- Приходы → `/arrivals`
- Расходы → `/departures`
- Погашения → `/payments`
- Партнеры → `/partners`
- Пользователи → `/management/users`

### Шаг 2: Найти запись
Найдите нужную запись в таблице.

### Шаг 3: Нажать кнопку удаления
В колонке "Действия" нажмите красную кнопку с иконкой корзины 🗑.

### Шаг 4: Подтвердить удаление
Появится диалоговое окно с запросом подтверждения:
```
Удалить эту запись? Это действие нельзя отменить!
```

Нажмите:
- **OK** - для подтверждения удаления
- **Отмена** - для отмены операции

### Шаг 5: Проверить результат
После удаления:
- Появится уведомление об успешном удалении
- Запись исчезнет из таблицы
- Страница автоматически обновится

## Что удаляется

### 📥 Приходы
При удалении прихода удаляется:
- Запись о поступлении товара
- Информация о вагоне
- Количество по документу и фактическое
- Примечания

**Влияние:**
- Уменьшается фактический остаток товара
- Изменяются отчеты по приходам
- Изменяются итоги вагонов

### 📤 Расходы
При удалении расхода удаляется:
- Запись о выводе товара
- Информация о клиенте (если был)
- Сумма продажи

**Влияние:**
- Увеличивается фактический остаток товара
- Уменьшается долг клиента (если был указан)
- Изменяются отчеты по расходам
- Изменяются долги клиентов

### 💰 Погашения
При удалении погашения удаляется:
- Запись об оплате
- Сумма в сомони и долларах
- Курс обмена

**Влияние:**
- Увеличивается долг клиента
- Изменяются отчеты по погашениям
- Изменяется карточка клиента

### 🤝 Партнеры
При удалении партнера удаляется:
- Запись об операции с партнером
- Сумма в сомони и долларах
- Курс обмена

**Влияние:**
- Изменяются отчеты по партнерам

### 👥 Пользователи
При удалении пользователя удаляется:
- Учетная запись пользователя
- Логин и пароль
- Права доступа

**Влияние:**
- Пользователь не сможет войти в систему
- История операций пользователя сохраняется
- Записи, созданные пользователем, остаются

## ⚠️ Важные предупреждения

### Необратимость
**Удаление необратимо!** Восстановить удаленную запись невозможно.

### Влияние на отчеты
Удаление записей влияет на:
- Фактические остатки
- Долги клиентов
- Все отчеты и сводки
- Итоговые суммы

### Влияние на баланс
При удалении:
- **Прихода** - уменьшается остаток товара
- **Расхода** - увеличивается остаток товара, уменьшается долг клиента
- **Погашения** - увеличивается долг клиента

### Рекомендации перед удалением

1. **Проверьте дважды**: Убедитесь, что удаляете правильную запись
2. **Запишите данные**: Сохраните информацию о записи (скриншот или запись)
3. **Проверьте влияние**: Подумайте, как удаление повлияет на отчеты
4. **Альтернативы**: Возможно, лучше создать корректирующую запись
5. **Уведомите команду**: Сообщите другим пользователям об удалении

## Когда использовать удаление

### Правильные случаи:
✅ Дублирующиеся записи
✅ Ошибочно созданные записи
✅ Тестовые данные
✅ Записи с неверными данными, которые нельзя исправить

### Неправильные случаи:
❌ Для исправления ошибок (лучше создать корректирующую запись)
❌ Для "очистки" старых данных (они нужны для истории)
❌ По просьбе обычного пользователя без проверки
❌ Массовое удаление без резервной копии

## Альтернативы удалению

### Вместо удаления можно:

1. **Создать корректирующую запись**
   - Для исправления ошибок в количестве
   - Для корректировки сумм
   - Сохраняет историю изменений

2. **Деактивировать пользователя**
   - Вместо удаления пользователя
   - Можно восстановить позже

3. **Добавить примечание**
   - Отметить запись как ошибочную
   - Объяснить ситуацию

## Безопасность

### Кто может удалять
- Только пользователи с ролью **admin**
- Обычные пользователи не видят кнопки удаления

### Логирование
- Все удаления должны логироваться
- Рекомендуется вести отдельный журнал удалений

### Резервные копии
- Делайте резервные копии БД перед массовым удалением
- Храните резервные копии минимум 30 дней

## Устранение проблем

**Ошибка при удалении:**
- Проверьте права доступа
- Убедитесь, что запись существует
- Проверьте связанные записи

**Случайно удалили запись:**
- Восстановите из резервной копии БД
- Создайте новую запись с теми же данными
- Обратитесь к системному администратору

**Не вижу кнопку удаления:**
- Проверьте, что вы вошли как администратор
- Обновите страницу
- Очистите кэш браузера

## Лучшие практики

1. **Минимизируйте удаления**: Удаляйте только когда действительно необходимо
2. **Документируйте**: Ведите журнал удалений с причинами
3. **Проверяйте**: Всегда проверяйте данные перед удалением
4. **Резервируйте**: Делайте резервные копии перед удалением
5. **Обучайте**: Объясняйте команде последствия удалений
6. **Ограничивайте**: Давайте права администратора только надежным людям

## Частые вопросы

**Q: Можно ли восстановить удаленную запись?**
A: Нет, только из резервной копии базы данных.

**Q: Что делать, если удалил не ту запись?**
A: Создайте новую запись с теми же данными или восстановите из резервной копии.

**Q: Почему нельзя редактировать вместо удаления?**
A: Функция редактирования будет добавлена в следующей версии.

**Q: Как удалить несколько записей сразу?**
A: В текущей версии только по одной. Для массового удаления обратитесь к администратору БД.

**Q: Удаляются ли связанные записи?**
A: Нет, удаляется только выбранная запись. Связанные записи остаются.
